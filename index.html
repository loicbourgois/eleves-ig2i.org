Élèves IG2I

Introduction

Salut à tou·te·s les IG2ien·ne·s !

Ce site a pour but de vous initier aux régles et conseils d'utilisation de votre organisation : eleves-ig2i. Si vous n'avez jamais utilisé Git, son utilisation vous semblera déroutante, voir compliquée au début, mais d'ici quelque temps vous remercirez les élèves qui ont voulu passer de Google Drive à GitHub !

Conventions de nommage

Pour un TP ou TD

ig2i-${promo}-${matière}-${année}-(tp|td)${numéro}-${titre}, avec :

  • ig2i : préfixe commun à l'ensemble des dépôts de l'organisation ;
  • ${promo} : votre promo, soit le1, le2, le3, le4, le5, la1, la2, la3 ;
  • ${matière} : diminutif figurant sur la plaquette, par exemple poo ;
  • ${année} : deuxième partie de l'année scolaire, soit 2018 pour 2017/2018 ;
  • ${numéro} : numéro du TP/TD ;
  • ${titre} : optionnel, titre du TP/TD.

Pour un CTP

ig2i-${promo}-${matière}-${année}-ctp

Pour un projet associé à une matière

ig2i-${promo}-${matière}-${année}-projet-${identifiant}, avec :

  • ${identifiant} : nom du projet ou de l'équipe.

Pour une matière-projet

ig2i-${promo}-${matière}-${année}-${identifiant}, avec :

  • ${matière} : matière parmis PINF, PIND ;
  • ${identifiant} : nom du projet ou de l'équipe.

Pour un outil lié à un événement

ig2i-${promo}-${événement}-${année}-${outil}, avec :

  • ${événement} : événement parmis JPO, JDN, RI, NDL5 ;
  • ${outil} : type de l'outil réalisé pour l'événement.

Quelques exemples

  • ig2i-le1-web1-2018-tp1 : TP n°1 de Web en première année initiale pour l'année scolaire 2017/2018 ;
  • ig2i-le2-pinf-2016-lespinpinfs : projet PINF non confidentiel réalisé par l'équipe de LE2 "Les Pinpinfs" ;
  • ig2i-l3-ndl5-2018-site : site web pour la Nuit des L5, géré par les LE3 et LA1.

Contenu

Pour qu'un nouveau dépôt soit valide, il contiendra à minima :

  • un fichier README.md qui expliquera brièvement en quoi consiste le dépôt ;
  • les sources.

Si le dépôt est associé à une matière, il devra aussi contenir :

  • le sujet du TP, TD, CTP ou projet, nommé comme suit : ${nom du dépôt}-sujet.pdf ;
  • le compte-rendu du TP, TD, CTP ou projet, nommé comme suit : ${nom du dépôt}-cr.(txt|pdf).

Par ailleurs, il est encouragé d'ajouter les fichiers suivants :

  • .gitignore : permet d'ignorer certains fichiers ;
  • .travis.yml : permet de lancer un build Travis et tester la validité du code.

Qualité

De manière à assurer l'homogénéité et la qualité dans les dépôts de l'organisation, les règles suivantes seront respectées :

  • la notation camelCase sera utilisée ;
  • les tabulations seront utilisées pour l'indentation ;
  • les espaces en fin de lignes (traling spaces en anglais) seront supprimés ;
  • seuls les fichiers sources seront traqués : l'utilisation d'un fichier .gitignore est encouragée.
